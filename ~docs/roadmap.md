## Coding Standards

### Formatting
- **Indentation:** Tabs (4-space width), never spaces
- **Brace Style:** K&R (opening brace on same line)
- **Hexadecimal:** Always lowercase
- **Encoding:** UTF-8
- **Line Endings:** CRLF (Windows standard)
- **File Endings:** Blank line at EOF
- **Trailing Whitespace:** Removed (except markdown line breaks)

### Documentation
- **Comment Everything:** All code should be well-documented
- **Chat Logs:** Maintain detailed conversation logs
- **Session Summaries:** Create summaries for each work session
- **Manual Prompts:** User maintains prompts log

### Git Workflow
- **Feature Branches:** Create branch for each significant feature
- **Issue Tracking:** All commits reference issues
- **Logical Commits:** Commit in logical, related batches
- **Branch Merging:** Merge feature branches when complete
