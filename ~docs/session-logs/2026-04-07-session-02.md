## Issues Completed

### #321 — Extract platform-specific header properties from SemanticAnalyzer

- Removed all 44 platform-specific header properties from `SemanticAnalyzer` (iNES, SNES, GB, Lynx, GBA, SPC)
- Created `IHeaderConfig` marker interface in `Poppy.Core.Arch`
- Created 6 concrete config classes: `NesHeaderConfig`, `LynxHeaderConfig`, `SnesHeaderConfig`, `GbHeaderConfig`, `GbaHeaderConfig`, `SpcHeaderConfig`
- Each profile uses `GetOrCreateConfig<T>()` pattern to create and store configs
- All 5 `SemanticAnalyzerExtensions` files updated to read from typed configs
- Delegated `.lorom`/`.hirom`/`.exhirom` from SA to `Wdc65816Profile.TryHandleDirective`
- Removed `HandleMemoryMapping` method from SA
- Added `HeaderConfig` property to SA (replaces `MemoryMapping` and `NesMapper`)
- Updated 2 test assertions
- Commit: `8a3c519`

### #323 — Extract 65816 M/X flag tracking from CodeGenerator fields

- Created `ProcessorState` class in `Poppy.Core.Arch`
- Added `CreateProcessorState()` factory and `TryHandleProcessorDirective()` to `ITargetProfile`
- Refactored `GetOperandSize` and `UpdateProcessorFlags` on `ITargetProfile` to take `ProcessorState?`
- `Wdc65816Profile` now owns all M/X flag logic (directives, REP/SEP, operand sizing)
- Removed `_accumulatorIs16Bit` and `_indexIs16Bit` fields from both `SemanticAnalyzer` and `CodeGenerator`
- SA's `AccumulatorIs16Bit`/`IndexIs16Bit` properties now delegate to `ProcessorState`
- SA's `TrackRepSep` delegates to `_profile.UpdateProcessorFlags`
- CG initializes `ProcessorState` in constructor and on platform switch
- Commit: `a353f28`
