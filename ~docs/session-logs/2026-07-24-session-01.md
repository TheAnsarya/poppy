# Session Log — 2026-07-24 Session 01

## Summary

Implemented multi-operand instruction support for the Poppy compiler, closing #246. This unblocks V30MZ ModR/M encoding (#204) and enables HuC6280 block transfer, 65816 block move, and future M68K/ARM multi-operand instructions.

## Work Completed

### #246 — Multi-Operand Instruction Support

**Phase 1 — AST (InstructionNode):**

- Changed `ExpressionNode? Operand` to `IReadOnlyList<ExpressionNode> Operands`
- Added backward-compatible `Operand` shim property (returns first operand or null)
- Two constructors: single-operand (existing callers unchanged) and multi-operand (new IReadOnlyList)

**Phase 2 — Parser:**

- Added `Is6502IndexRegisterAfterComma()` peek-ahead method for comma disambiguation
- Only `x`/`y`/`s` as standalone single-letter identifiers trigger 6502 index register handling
- Multi-operand architectures (V30MZ `ax`/`bx`, M68K `d0`/`a0`, ARM `r0`) use multi-letter registers and are never ambiguous
- `ParseInstruction()` now collects additional comma-separated operands after the first
- Updated test: `Parse_InvalidIndexRegister_ReportsError` → `Parse_NonIndexRegisterAfterComma_ParsesAsMultiOperand`

**Phase 3 — Encoder Context:**

- Added `ResolvedOperand` record struct (Identifier + Value)
- Extended `SpecialInstructionContext` with optional `AdditionalOperands` list
- `CodeGenerator` resolves all additional operands and passes them to encoders
- `SemanticAnalyzer` now visits ALL operands (not just the first)

**9 new parser tests covering:**

- Two-operand register-register (`mov ax, bx`)
- Two-operand register-immediate (`cmp ax, #$10`)
- Three-operand block transfer (`tii $2000, $3000, $100`)
- 65816 block move (`mvn $00, $7e`)
- 6502 indexing regression tests (,x / ,y / ,s still work)
- Backward-compat Operand shim
- Implied mode empty Operands
