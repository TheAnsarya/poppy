# Session Log — 2026-04-04 Session 04

## Summary

Discovered and fixed a critical compiler bug where `.target` directives in source files were being ignored by the CodeGenerator. This caused all non-NES/SNES/GB custom test ROMs to fail assembly. Also expanded the CLI `--target` flag from 3 to 12 architectures.

## Issues

| Issue | Action | Description |
|-------|--------|-------------|
| [#249](https://github.com/TheAnsarya/poppy/issues/249) | Created & Closed | `.target` directive not propagated to CodeGenerator |

## Root Cause

In `Program.cs`, the CodeGenerator constructor received `options.Target` (default `MOS6502`) instead of `analyzer.Target` which is resolved from `.target` directives in source files. The `SemanticAnalyzer` correctly resolved `.target ws`, `.target lynx`, `.target pce`, etc., but the resolved target was never passed to the CodeGenerator. The adjacent CDL generator line already correctly used `analyzer.Target`.

## Changes

### `src/Poppy.CLI/Program.cs`

1. **Critical fix**: Changed CodeGenerator construction to use `analyzer.Target` via `resolvedTarget` variable instead of `options.Target`
2. **CLI expansion**: Expanded `--target` CLI flag from 3 to 12 architectures:
	- 6502/nes, 6507/atari2600, 65sc02/lynx, 65816/snes, sm83/gb
	- m68000/genesis/md, z80/sms, v30mz/ws/wonderswan
	- arm7tdmi/gba, spc700, huc6280/tg16/pce, f8/channelf
